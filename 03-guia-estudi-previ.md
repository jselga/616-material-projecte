# Guia breu estudi previ Projecte Final DAW
## Abans de començar a programar

L'estudi previ serveix per definir el projecte abans d'implementar-lo. Ha de demostrar que l'equip ha pensat el sistema abans de programar.

---




## 1 Descripció del sistema i context

Explicar quin problema resol l'aplicació.

Cal descriure:

- Quin és l'objectiu de l'aplicació
- Quin tipus d'usuaris tindrà
- En quin context s'utilitzarà
- Quina necessitat resol

No cal un text molt llarg, però sí una explicació clara del sistema.

---

## 2 Requisits del sistema

### Requisits funcionals

Descriuen **què pot fer el sistema**.

Exemples:

- Registrar usuaris
- Crear i gestionar contingut
- Cercar informació
- Gestionar dades
- Sistema de missatges o notificacions

És recomanable agrupar-los per funcionalitats.

### Requisits no funcionals

Descriuen **com ha de funcionar el sistema**.

Exemples habituals:

- Seguretat
- Rendiment
- Usabilitat
- Disponibilitat
- Compatibilitat amb navegadors
- Compliment de normativa de dades

---

## 3 Model conceptual (model de dades)

Definir les entitats principals del sistema.

Cal incloure:

- Diagrama d'entitats o classes
- Relacions entre entitats

També és recomanable descriure breument cada entitat.

Exemple:

Usuari → representa les persones que utilitzen el sistema Publicació → contingut creat pels usuaris Missatge → comunicació entre usuaris

---

## 4 Model de negoci (casos d'ús)

Cal identificar:

- actors del sistema
- accions principals que poden realitzar

Actors habituals:

- usuari
- administrador
- visitant

Es recomana incloure un **diagrama de casos d'ús** que representi les funcionalitats principals.

---

## 5 Disseny inicial de la interfície

Esbossos de les pantalles principals de l'aplicació.

No cal que siguin dissenys finals, poden ser:

- wireframes
- mockups
- esquemes de pantalles

Han d'incloure com a mínim:

- pàgina principal
- llistats
- formularis
- pàgines de detall

---

## 6 Tecnologies utilitzades

Explicar les tecnologies que s'utilitzaran al projecte.

Exemple:

Frontend

- React / Next / Vue
- HTML
- CSS / Tailwind / Bootstrap

Backend

- Laravel
- Node / Express

Base de dades

- MySQL
- MongoDB

També es pot explicar breument per què s'ha escollit cada tecnologia.

---

## 7 Planificació del projecte

Cal explicar com s'organitzarà el desenvolupament.

Es recomana incloure:

- tasques principals del projecte
- estimació d'hores
- planificació temporal

Es pot representar amb:

- diagrama de Gantt
- tauler de Trello
- llista de fases del projecte
