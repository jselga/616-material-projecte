# Proposta de requisits del Projecte Final DAW 

## Objectiu del projecte

El projecte final del cicle ha de permetre que l'alumnat demostri la seva **maduresa com a desenvolupadors d'aplicacions web**, integrant coneixements adquirits al llarg dels diferents mòduls del cicle.

L'objectiu és que els estudiants desenvolupin una **aplicació web completa** que combini frontend, backend, base de dades i desplegament real.

Aquest projecte ha de servir com a exercici integrador on es posin en pràctica competències tècniques, capacitat de planificació, treball en equip i bones pràctiques de desenvolupament.

---

# Característiques generals del projecte

El projecte haurà de complir les següents característiques:

- Desenvolupament d'una **aplicació web completa (full‑stack)**.
- Tema del projecte **lliure**, però amb una funcionalitat clara i coherent.
- Integració de **frontend, backend i base de dades**.
- Implementació de **diferents rols d'usuari dins l'aplicació**.
- Ús de **control de versions amb Git**.
- Documentació del projecte.
- Desplegament de l'aplicació en un servidor accessible.

L'aplicació ha de mostrar un nivell de **qualitat tècnica i organització del codi propi d'un projecte professional bàsic**.

---

# Arquitectura del projecte

El projecte haurà d'incloure com a mínim:

## Frontend

- Interfície d'usuari interactiva
- Consum de serveis o API
- Formularis i validació

## Backend

- API o sistema de rutes
- Lògica de negoci
- Validacions al servidor
- Gestió d'usuaris i autenticació

## Base de dades

- Persistència de dades
- Model de dades coherent
- Relacions entre entitats

---

# Tecnologies

Els estudiants hauran d'utilitzar **tecnologies treballades durant el curs** combinades així com incorporar tecnologies noves.

Exemples de combinacions possibles:

- Laravel + React
- Laravel + Vue
- Stack MERN (MongoDB, Express, React, Node)
- Next.js
- Astro


També es permeten altres combinacions equivalents sempre que es mantingui una arquitectura web coherent.

---

# Interfície i experiència d'usuari

La interfície de l'aplicació ha d'estar **cuidada i usable**.

Es poden utilitzar frameworks d'estil com:

- Bootstrap
- TailwindCSS
- altres frameworks equivalents

Es valorarà especialment:

- coherència visual
- navegació clara
- disseny responsive
- qualitat de l'experiència d'usuari
- accessiblitat i usabilitat

---

# Organització del desenvolupament

El projecte haurà de començar amb una **fase de planificació prèvia** que inclogui:

- anàlisi del problema
- definició de requisits
- model de dades
- disseny inicial de la interfície
- estimació de tasques

Aquesta planificació haurà d'estar **ben estructurada i documentada**.

---

# Bones pràctiques de desenvolupament

El projecte haurà d'aplicar bones pràctiques com:

- control de versions amb Git
- estructura clara del repositori
- repositori compartit amb integrants del grup i els professors
- separació de responsabilitats (frontend / backend)
- codi llegible i mantenible

---

# Desplegament

L'aplicació haurà de ser desplegable tant en un servidor web com per mitjà de contenidors docker

El projecte haurà d'incloure:

- documentació de desplegament
- configuració necessària per executar l'aplicació
- accés públic o demostrable de l'aplicació

---

# Finalitat educativa

Aquest projecte ha de permetre a l'alumnat demostrar que és capaç de:

- dissenyar una aplicació web
- implementar una arquitectura full‑stack
- treballar amb diferents tecnologies
- organitzar el desenvolupament d'un projecte real
- desplegar una aplicació funcional
